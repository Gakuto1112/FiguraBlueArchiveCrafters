# README生成ツール

このツールは本レポジトリ（Figura Blue Archive Crafters, FBAC）のREADMEのマークダウンドキュメント および 配布データに含めるREADMEテキストファイルをテンプレートから作成するツールです。

## テンプレートファイル

"[templates/](./templates/)"ディレクトリにREADMEのテンプレートとなるファイルがあります。
"en"は英語のテンプレート、"jp"は日本語のテンプレートです。
また、`.md`はレポジトリのREADME用のマークダウンファイル、`.txt`は配布データ用のテキストファイルです。

### テンプレートマークダウン内のアンカータグ

テンプレートマークダウンファイル（"[en.md](./templates/en.md)"および"[jp.md](./templates/jp.md)"）には、マークダウンコメント形式（`<!-- -->`）で記入されたアンカータグがあります。
これらのアンカータグはR、README生成ツールにおいて目印として利用されます。

アンカータグは次のような形式で記入されています：

```md
<!-- ${アンカータグ名} --->
```

アンカータグはその行に単体で配置されていることを想定としています。
もしアンカータグと同じ行にテキストがあった場合は、そのテキストが無視される可能性があります。

以下に、アンカータグとその役割を示します：

| アンカータグ名 | 説明 |
| --- | --- |
| DESCRIPTION_START | レポジトリの説明文の開始タグ <br> `DESCRIPTION_END`とセットで使用する。このブロックはREADMEのテキストファイルを作成する際に利用される。 |
| DESCRIPTION_END | レポジトリの説明文の終了タグ <br> `DESCRIPTION_START`とセットで使用する。 |
| USAGE_START | アバターの使用方法の開始タグ <br> `USAGE_END`とセットで使用する。このブロックはREADMEのテキストファイルを作成する際に利用される。 |
| USAGE_END | アバターの使用方法の終了タグ <br> `USAGE_START`とセットで使用する。 |
| NOTES_START | アバターの注意事項の開始タグ <br> `NOTES_END`とセットで使用する。このブロックはREADMEのテキストファイルを作成する際に利用される。 |
| NOTES_END | アバターの注意事項の終了タグ <br> `NOTES_START`とセットで使用する。 |
| CREATION_STATUS_DONE | このタグの行が「作成完了済みキャラクターのリスト」で置換されます。 |
| CREATION_STATUS_IN_PROGRESS | このタグの行が「作成中のキャラクターのリスト」で置換されます。 |
| CREATION_STATUS_PLANNED | このタグの行が「作成予定のキャラクターのリスト」で置換されます。 |
| CREATION_STATUS_REQUESTED | このタグの行が「作成リクエストを受けたキャラクターのリスト」で置換されます。 |

### テンプレートテキスト内のアンカータグ

テンプレートテキストファイル（"[en.txt](./templates/en.txt)"および"[jp.txt](./templates/jp.txt)"）には、`${アンカー名}`のような形式で記入されたアンカータグがあります。
これらのアンカータグは、README生成ツールにおいて目印として利用されます。

以下に、アンカータグとその役割を示します：

| アンカータグ名 | 説明 |
| --- | --- |
| TAG_NAME | 引数から入力されたタグ名（`v3.0.0`など）で置換されるアンカータグです。 |
| RELEASE_DATE | 引数から入力された日付（バージョンのリリース日）で置換されるアンカータグです。 |
| DESCRIPTION | このタグの行がレポジトリの説明文（テンプレートマークダウンファイル内の`<!-- DESCRIPTION_START --->`から`<!-- DESCRIPTION_END --->`までの部分）に置換されます。 |
| USAGE | このタグの行がアバターの使用方法（テンプレートマークダウンファイル内の`<!-- USAGE_START --->`から`<!-- USAGE_END --->`までの部分）に置換されます。 |
| NOTES | このタグの行がアバターの注意事項（テンプレートマークダウンファイル内の`<!-- NOTES_START --->`から`<!-- NOTES_END --->`までの部分）に置換されます。 |

テンプレートマークダウンファイルの一部分に置き換えるタグについて、置き換える過程で、マークダウン構文（太字やURL）などは削除され、通常の文章として挿入されます。

## 作成状況

テンプレートマークダウンファイルに含まれる作成状況の置換内容は、"[creation_status.json](./creation_status.json)"を基に作成されます。
このjsonファイルの構造は次の通りです：

```
📃 creation_status.json
├ 📁 done[]
│ └ 📁 character_entry{}
├ 📁 in_progress[]
│ └ 📁 character_entry{}
├ 📁 planned[]
│ └ 📁 character_entry{}
└ 📁 requested[]
  └ 📁 character_entry{}
```

（🔢: 数字型、🔠: 文字列型、▶️: ブーリアン型、📁[]: 配列型、📁{}: 辞書型、(): 任意のフィールド）

`done`内には作成済みのキャラクターのエントリーを、`in_progress`内には作成中のキャラクターのエントリーを、`planned`内には作成予定のキャラクターのエントリーを、`requested`内には作成をリクエストされたキャラクターのエントリーを、それぞれ配列形式で記入します。
出力されるマークダウンリストの項目の順番は、この配列の順番と同じなので、作成済みのキャラクターのエントリーは作成した順、作成予定・作成リクエストのキャラクターのエントリーは優先度順で並べます。

各キャラクターエントリーの構造は次の通りです：

```
📁 character_entry{}
├ 📁 character_name{}
│ ├ 📁 first_name{}
│ │ ├ 🔠 en
│ │ └ 🔠 jp
│ └ 📁 last_name{}
│   ├ 🔠 en
│   └ 🔠 jp
├ 📁 (costume_name{})
| ├ 🔠 en
│ └ 🔠 jp
└ 🔢 (issue_number)
```

（🔢: 数字型、🔠: 文字列型、▶️: ブーリアン型、📁[]: 配列型、📁{}: 辞書型、(): 任意のフィールド）

| 項目 | 型 | 説明 |
| --- | --- | --- |
| character_name | オブジェクト | キャラクターの氏名 |
| character_name.first_name | オブジェクト | キャラクターの下の名前 |
| character_name.first_name.en | 文字列 | キャラクターの下の名前の英名 |
| character_name.first_name.jp | 文字列 | キャラクターの下の名前の日本語名 |
| character_name.last_name | オブジェクト | キャラクターの下の名前 |
| character_name.last_name.en | 文字列 | キャラクターの上の名前の英名 |
| character_name.last_name.jp | 文字列 | キャラクターの上の名前の日本語名 |
| costume_name | オブジェクト or null | キャラクターの衣装名 <br> デフォルト衣装は`null`にする。 |
| costume_name.en | 文字列 | キャラクターの衣装の英名 |
| costume_name.jp | 文字列 | キャラクターの衣装の日本語名 |
| issue_number | 数値 or null | このエントリーに関するissue番号 <br> ない場合は`null`にする。 |

もしエントリーが0の項目がある場合は、マークダウンリストの代わりにその旨のメッセージのテキストが挿入されます。

## 実行手順

本生成ツールの実行にはPythonのバージョン管理ツールである[uv](https://docs.astral.sh/uv/)が必要です。
また、手順内にあるコマンド例はMac/Linux準拠になります。

1. ワーキングディレクトリを"/src/readme_scripts"に設定します。

   ```sh
   cd <path_to_repository_root_directory>/readme_scripts/
   ```

2. Python及び依存パッケージのインストールをします。
   以下のコマンドを実行するだけでインストールできます。

   ```sh
   uv sync
   ```

3. ビルドスクリプトを実行します。
   テキストファイルを生成する場合は、引数にタグ名とリリース日を渡す必要があります。
   デフォルトパスの場合、`../`に生成済みドキュメントが出力されます。

   ```sh
   uv run generate_readme_md.py # マークダウンファイルの生成の場合
   uv run generate_readme_txt.py ${TAG_NAME} ${RELEASE_DATE} # テキストファイルの生成の場合
   ```

## 引数

本生成ツールには（オプション）引数を用意しています。
必須引数については指定順に記載しています。

### generate_readme_md.py

| 引数名 | 追加引数 | 説明 | 必須引数か？ |
| --- | --- | --- | --- |
| -h, --help | なし | ビルドツールの説明を出力します。 | いいえ |
| -t, --template-dir | テンプレートディレクトリまでのパス | テンプレートを読み込むディレクトリを指定します。この引数を指定しない場合は`./templates/`になります。 | いいえ |
| -o, --dist-dir | 出力先ディレクトリまでのパス | 出力先のディレクトリを指定します。この引数を指定しない場合は`../`になります。 | いいえ |
| -l, --colored | なし | 標準出力に色を付けます。ログ出力などの制御文字がそのまま出力される場合はオフにしてください。 | いいえ |

### generate_readme_txt.py

| 引数名 | 追加引数 | 説明 | 必須引数か？ |
| --- | --- | --- | --- |
| tag_name | リリースタグの名前 <br> （`v3.0.0`など） | リリースタグの名前を入力します。 | はい |
| release date | リリース日 <br> `2026-04-26T15:00:00Z`のような形式で渡します。 | リリース日を入力します。 | はい |
| -h, --help | なし | ビルドツールの説明を出力します。 | いいえ |
| -t, --template-dir | テンプレートディレクトリまでのパス | テンプレートを読み込むディレクトリを指定します。この引数を指定しない場合は`./templates/`になります。 | いいえ |
| -o, --dist-dir | 出力先ディレクトリまでのパス | 出力先のディレクトリを指定します。この引数を指定しない場合は`../`になります。 | いいえ |
| -l, --colored | なし | 標準出力に色を付けます。ログ出力などの制御文字がそのまま出力される場合はオフにしてください。 | いいえ |
